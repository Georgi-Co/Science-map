# Карта научно-исследовательских работ РГЭУ (РИНХ)

Дипломный проект бакалавра. Веб-приложение для отображения и поиска научных работ студентов и преподавателей Ростовского государственного экономического университета (РИНХ).

## Описание проекта

Веб-приложение "Карта НИР" представляет собой платформу для публикации, поиска и просмотра научно-исследовательских работ студентов и преподавателей РГЭУ (РИНХ). Проект создан в рамках дипломной работы бакалавра и предоставляет удобный интерфейс для доступа к научным материалам университета.

## Основные функции

- Просмотр списка всех научных работ
- Поиск по ключевым словам
- Фильтрация по факультетам, научным областям и направлениям
- Просмотр полной информации о научной работе
- Просмотр информации об авторах научных работ
- Адаптивный дизайн для различных устройств
- Версия для слабовидящих
- Поддержка двух языков (русский/английский)

### Структура данных

#### Article (Научная работа)
- Title (Заголовок)
- Content (Содержание)
- Slug (Уникальный идентификатор URL)
- Publication (Дата публикации)
- Faculty (Факультет)
- Media (Медиафайлы)
- Authors (Авторы) - связь many-to-many
- Tags (Теги) - связь one-to-many

#### Author (Автор)
- Name (Имя)
- Info (Информация)
- Avatar (Фото)
- Email (Электронная почта)
- Slug (Уникальный идентификатор URL)
- Articles (Научные работы) - связь many-to-many

#### Tag (Тег)
- Name (Название)
- Slug (Уникальный идентификатор URL)

## API Endpoints

- `http://localhost:1337/api/articles?populate=*` - Получение списка научных работ
- `http://localhost:1337/api/authors?populate=*` - Получение списка авторов
- `http://localhost:1337/api/tags?populate=*` - Получение списка тегов

## Установка и запуск

1. Установите зависимости Strapi:
   ```bash
   cd admin-strapi
   npm install
   ```

2. Запустите Strapi сервер:
   ```bash
   npm run develop
   ```

3. Откройте в браузере `http://localhost:1337/admin` для доступа к панели администратора

4. Для просмотра фронтенда откройте файл `html/index.html` в браузере

## Входные и выходные данные проекта

### Входные данные
- **Научные работы**: заголовок, содержание (блоки), дата публикации, факультет (выбор из предопределённого списка), научная область, научное направление, медиафайлы (изображения, видео, аудио, документы), связь с авторами и тегами.
- **Авторы**: имя, краткая информация, аватар (изображение), электронная почта.
- **Теги**: название тега.
- **Конфигурационные данные**: настройки Strapi, роли пользователей, права доступа.

Данные вводятся через административную панель Strapi (`http://localhost:1337/admin`) либо импортируются через API.

### Выходные данные
- **Веб-интерфейс**: HTML-страницы (главная, детальная статья, страница автора) с применёнными CSS-стилями и JavaScript-логикой.
- **API-ответы**: JSON-структуры, возвращаемые эндпоинтами Strapi, содержащие массивы статей, авторов, тегов с популяцией связанных данных.
- **Динамический контент**: отфильтрованные и отсортированные списки работ, результаты поиска, пагинация.
- **Дополнительные выходы**: стили для слабовидящих (высококонтрастный режим, увеличение шрифта), локализованные тексты (русский/английский).

## База данных SQLite

Проект использует SQLite в качестве базы данных по умолчанию для среды разработки. Это облегчает развёртывание и тестирование без необходимости настройки внешней СУБД.

### Конфигурация
- Файл базы данных: `admin-strapi/.tmp/data.db`
- Настройки подключения определены в `admin-strapi/config/database.js` (клиент `sqlite`).
- База данных автоматически создаётся при первом запуске Strapi.

### Структура
Основные таблицы, соответствующие моделям контента:

1. **articles** – хранит научные работы.
   - Поля: `id`, `Title`, `Content` (JSON), `Slug`, `Publication`, `Faculty`, `ScienceArea`, `ScienceDirection`, `created_at`, `updated_at`, `published_at`.
   - Связи: многие-ко-многим с `authors`, один-ко-многим с `tags`.

2. **authors** – хранит авторов.
   - Поля: `id`, `Name`, `Info`, `Avatar` (ID медиафайла), `Email`, `Slug`, `created_at`, `updated_at`, `published_at`.

3. **tags** – хранит теги.
   - Поля: `id`, `Name`, `Slug`, `created_at`, `updated_at`, `published_at`.

4. **Связующие таблицы** для отношений many-to-many (например, `articles_authors_links`).

5. **Служебные таблицы Strapi**: `strapi_users`, `strapi_roles`, `strapi_permissions`, `strapi_webhooks`, `strapi_i18n_locale` и др.

### Миграции
Strapi автоматически генерирует и применяет миграции при изменении схемы контента. Файлы миграций хранятся в `admin-strapi/database/migrations/`.

### Резервное копирование
Рекомендуется периодически копировать файл `data.db` для сохранения данных. В production-среде рекомендуется переключиться на PostgreSQL или MySQL.
